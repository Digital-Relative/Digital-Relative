Stripe VAT handling
One toggle in the Stripe dashboard

Selling to UK consumers requires correct VAT collection. Stripe Tax automates this entirely — it calculates the correct VAT rate per country, adds it to the checkout, and produces compliant invoices. Without it you may be under-collecting tax as you scale. This is one toggle in the Stripe dashboard under Billing → Tax.

ICO registration
£40 at ico.org.uk — update footer from "pending"

Any UK business that processes personal data must register with the ICO. The registration costs £40/year for small organisations. Once done, update the ICO number in the footer, privacy policy, and terms. Non-registration is a legal offence regardless of whether anyone complains.

Push notifications via service worker
Check-in reminder before it becomes overdue

Email check-in reminders only fire when already overdue. A push notification 3 days before the deadline — "Your Digital Relative check-in is due in 3 days" — prevents false triggers entirely. Service workers work in Chrome, Edge, and Firefox on desktop and Android. iOS requires a PWA install prompt. Low effort, significant reduction in accidental switch triggers which is the #1 support issue for this type of product.

📲
Mobile app (React Native wrapper)
Face ID / fingerprint vault unlock, push notifications

The web app is fully responsive but lacks native mobile capabilities. A React Native wrapper using the existing Supabase auth would get 80% of the way there. The two unique mobile features that justify native: biometric vault unlock (Face ID or fingerprint replaces the vault PIN on mobile) and push notifications for check-in reminders before they become overdue. Offline reading of vault data is also possible with encrypted local storage. Apple's App Store review is the main timeline risk — allow 4–6 weeks for review.

Structured data fields per category
Dedicated fields for policy numbers, sort codes, account numbers

Currently all entries use a generic username/password/notes model. A bank account would benefit from dedicated fields: sort code, account number, IBAN. An insurance policy: policy number, insurer, sum assured, renewal date. A pension: provider, policy reference, scheme name. Structured fields make data more useful for beneficiaries and enable smarter features (e.g. auto-populating bereavement letter templates with the right reference numbers).

Smarter vault health score
Beyond category coverage

The current completeness score counts covered categories. It could be much smarter: flag entries with no username (incomplete record), flag entries whose passwords haven't been updated in over a year (suggest rotation), flag entries with no bereavement contact matched (prompt to add), flag unconfirmed beneficiary invites, flag missing executor designation. Each flag becomes an actionable item rather than a percentage. This is the difference between "73% complete" and "3 things to fix."

Solicitor and IFA referral programme
The highest-trust acquisition channel in this category

Solicitors and Independent Financial Advisers routinely ask clients "have you organised your digital accounts?" and currently have no answer to give them. A simple referral programme where professionals earn a small commission per referred paying customer would be high-value on both sides. The implementation is just a referral code on signup and a tracking table. The sales effort is visiting 10 local solicitors and IFAs with a one-pager.

White-label is the next step: the same product under a firm's brand. This is a significantly larger technical and commercial investment but potentially a much larger revenue stream — firms pay per seat rather than per user, and churn is near zero.

🏦
Open Banking integration
Auto-detect accounts — never have to manually add banking entries

Under Open Banking (PSD2 in the UK), users can grant read-only access to their financial accounts via TrueLayer or Plaid. Digital Relative could use this to auto-detect which banks, investment platforms, and credit cards a user has — and pre-populate vault entries. This removes the biggest friction in vault setup ("I'm not sure which accounts I have"). The data would be encrypted immediately and the Open Banking connection could be revoked. The regulatory complexity of being an Open Banking Third Party Provider is the main barrier.

🔐
Hardware key (WebAuthn / FIDO2) support
YubiKey as vault unlock method

WebAuthn (the Web Authentication API) is now widely supported and allows hardware security keys like YubiKey to authenticate users without a PIN or password. For high-security users — particularly those in regulated professions or with large estates — a hardware key as the vault unlock method is significantly more phishing-resistant than a PIN. The technical implementation uses the browser's native WebAuthn API, which can be integrated with Supabase's existing auth system.
