Feature Overview

A complete reference of everything in the platform — for users, beneficiaries, executors, and admin.

v99 · 24 May 2026 · Confidential

Vault owner experience

Everything a paid user can do from login to legacy planning.

Vault
🔐

Encrypted vault entries

Store accounts across 13 categories: Banking, Email, Investments, Property, Insurance, Subscriptions, Government, Social Media, Utilities, Medical, Legal, Secure Note, and Other. Every field encrypted with AES-256-GCM before leaving the device.

📝

Secure notes New

A full-size encrypted text area for sensitive information that doesn't fit account-credential format — passport numbers, NI number, will location, PIN codes, insurance policy details, or any private note.

📅

Expiry tracking

Set an expiry date on any entry (passports, policies, licences, contracts). Choose reminder windows of 7, 14, 30, 60, or 90 days. Dashboard shows a banner of expiring items.

🏠

Address field with postcode lookup

Each entry can store a postal address. UK postcode lookup via GetAddress.io with manual entry fallback.

🏢

Bereavement contacts

100+ UK companies pre-loaded with verified bereavement phone numbers, URLs, and process notes. Select a company when adding an entry and this information is automatically saved — visible to beneficiaries after death.

🕐

Version history

Every save creates a snapshot (up to 3 previous versions per entry) including title, category, all encrypted fields, and address. Restore any previous version with one click.

⬆

CSV import New

Import from 1Password, Bitwarden, LastPass, Chrome, Edge, or any generic CSV. Auto-detects format. Preview and select entries before importing. Plan limit enforced. Passwords encrypted client-side before saving.

🔗

Shared links

Generate a secure link to share a single vault entry outside the platform — for a solicitor, accountant, or trusted contact. Optional PIN protection, view count tracking, and owner notification on access.

Documents
📄

Document storage

Upload important documents (up to 25MB each) across 9 categories: Identity, Property, Legal, Financial, Insurance, Medical, Employment, Vehicle, Other. Files stored encrypted in UK Supabase Storage.

👁

In-browser preview New

PDFs render in a sandboxed iframe. Images (PNG, JPG, GIF, WebP) render inline. SVG excluded for security. Download remains available from preview. URL cached for repeat opens.

🔗

Document sharing

Share any document via a signed URL — useful for sending a will or insurance policy to a solicitor without giving them full vault access.

Beneficiary management
👥

Named beneficiaries

Add beneficiaries by name, email, and relationship. Free plan: 1. Single: 3. Couples: 5. Each beneficiary receives an invite email and the vault owner receives a notification email with a security warning.

🏷

Beneficiary groups New

Assign an optional group label to each beneficiary (e.g. "Family", "Legal", "Friends"). Filter tabs appear when multiple groups exist. Gold badge shown on each card.

🔒

Access levels

Three levels per beneficiary: Full access (all vault categories), Read only (view without passwords), Specific categories. Enforced server-side — passwords never exposed regardless of access level.

📋

Access requirements

Three verification requirements per beneficiary: Death certificate (Onfido-verified), ID only (identity check, no death certificate), Trust only (immediate access on invite acceptance — for very trusted people).

⭐

Executor designation

Designate one beneficiary as executor. Executors get a dedicated executor tab in the beneficiary portal with a 26-task estate administration checklist.

📨

Resend invite

Resend the invitation email to any beneficiary who hasn't confirmed, with a 1-hour rate limit per beneficiary to prevent spam.

After I'm Gone guide
✉️

Personal message

Write a private message to your beneficiaries — shown to them in their portal after access is granted. Stored encrypted.

🌸

Funeral wishes

Record burial or cremation preference, music choices, readings, funeral home with address lookup, and any other wishes. Displayed to beneficiaries alongside the personal message.

📋

Getting started guide

8-step onboarding checklist covering: first vault entry, add beneficiary, designate executor, record will location, complete After I'm Gone guide, write personal message, set check-in frequency, and set up MFA. Tracks real completion from the database.

Check-in and dead man's switch
✅

Configurable check-in

Set a check-in frequency of 14, 30, 60, or 90 days. Check in from the dashboard with one click (60-second idempotency guard prevents double-check-in). Dashboard shows next due date and countdown.

⚠️

Overdue reminders

If check-in is missed, email reminders are sent at 3, 7, and 14 days overdue. If an SMS-capable phone number is provided, an SMS reminder is sent alongside each email.

🔔

Switch trigger

After the full frequency period passes without a check-in, beneficiaries receive email notifications to begin the access process. Vault owner can check in at any time to halt the process.

Security
🔑

Two-credential architecture

Login password controls account access. Vault PIN encrypts vault contents (PBKDF2 600k iterations). Changing your login password does not affect vault encryption. Your PIN never leaves your device.

🛡

Multi-factor authentication

Mandatory for email/password accounts. TOTP authenticator app (primary) or email OTP (fallback). MFA unenrolment requires email OTP re-verification. Lost device: identity verification then service-role reset.

🔮

Recovery codes New

Generate 8 one-time vault PIN recovery codes. Each independently encrypts the PIN (PBKDF2 100k). If you forget your vault PIN, redeem a code to regain access. Codes are marked used on redemption. Fetch count tracks potential exfiltration.

🎭

Duress PIN

Set a secondary PIN that shows a decoy vault. When used, silently alerts the vault owner and Digital Relative's security team by email. No visible indicator that duress mode is active.

📱

Trusted device

Optionally skip PIN entry on a trusted device. PIN encrypted with a device-specific key (PBKDF2 from a localStorage token). Clearing browser storage revokes trust. Manage trusted devices in Settings.

🔒

Auto-lock

30-minute inactivity lock on the vault. 8-hour absolute session limit (applies even on trusted devices). Re-entering PIN or trusted device auto-unlock resumes the session.

Settings
🌍

Language

Interface available in English, Polish, Urdu, and Arabic. Arabic applies right-to-left layout automatically. Language preference stored per user.

📊

Audit log

Recent account activity including sign-ins, vault changes, beneficiary additions, and MFA events. Visible in Settings under Recent activity.

💻

Device log

All recent sign-ins with device type, approximate location, and timestamp. New device triggers an alert email. Review and monitor from Settings.

📤

GDPR data export

Export all your personal data as a JSON file with a SHA-256 integrity hash. Includes vault entry metadata (ciphertext), beneficiaries, audit log, and preferences. Available from Settings.

🗑

Account deletion

Two-step deletion: type DELETE, then re-enter your login password (OAuth users skip password step). Permanently deletes vault-files and death-certificates storage buckets, anonymises audit log.

📱

SMS reminders New

Optional mobile number (E.164 format) for SMS check-in reminders alongside email. Validated client-side and server-side. Twilio only used if a number is provided.

🔗

Shared links management

View, revoke, and track all shared links from Settings. See view counts and last access time.

👩‍💑

Couples vault

Invite a partner to share the couples plan. Each partner retains a fully separate vault with its own PIN. Couples plan covers both vaults (5 beneficiaries, 5GB storage). Separation flow preserves both vaults.

Emergency access
🏥

Death certificate submission

Beneficiaries can submit a death certificate for Onfido verification to trigger access. File validated by magic bytes. Max 25MB.

⏳

48-hour hold on all access

Every emergency access request (regardless of verification method) is held for 48 hours. The vault owner is notified and can deny the request if still alive. Hold enforced server-side by the scheduler — cannot be bypassed from the client.

Beneficiary portal

What a nominated beneficiary sees and can do after receiving an access link.

Access and identity
🔗

Token-based access

Beneficiaries access the portal via a personal invite link. The link contains a one-time-style token. No account or login required.

🪪

Identity verification

Powered by Onfido. Upload a passport, driving licence, or national ID. On verification, Tier 2 access unlocks — revealing account usernames and notes in addition to bereavement contacts.

⏰

Session security

30-minute inactivity timeout with a clear "Session expired" screen. All vault content clears on timeout. No persistent storage of vault data in the browser.

🌍

Geographic access alert

If a beneficiary accesses the portal from outside the UK, the vault owner receives an automatic email alert (detected via Cloudflare's IP geolocation headers).

What to do tab
⚡

Immediate tasks

Pre-populated checklist: Register the death (within 5 days), Use Tell Me Once (notify HMRC, DVLA, DWP, council in one step), Arrange the funeral. Links to gov.uk resources.

📅

First week tasks

Contact a solicitor about probate, plus all vault entries categorised as Banking, Legal, or Other — with their bereavement contact numbers.

🗓

First month tasks

Vault entries categorised as Investments, Insurance, Utilities, Property, Medical, and Digital — each with bereavement contacts where pre-loaded.

🔗

Helpful service links

Direct links to: Tell Me Once, Death Notification Service, Bereavement Register, Cruse Bereavement Support, Citizens Advice, and GOV.UK Probate.

Their guide tab
💛

Personal message

The vault owner's personal message to their family — displayed prominently in a gold card.

🌸

Funeral wishes

Burial or cremation preference, music, readings, funeral home details, and any other wishes the owner recorded.

Letter templates tab
✉️

Bereavement letter template

Pre-written professional letter for notifying institutions of a death. Auto-populated with the deceased's name and the beneficiary's name. One-click copy to clipboard.

Emergency access
📋

Death certificate submission

Beneficiaries can submit a death certificate to request vault access if the dead man's switch hasn't triggered naturally. The request enters a 48-hour hold. The vault owner can deny during this period.

Executor dashboard

The executor tab — visible only to the beneficiary designated as executor by the vault owner.

Overview
📊

Progress tracking

Overall completion percentage bar showing how many of the 26 tasks are marked Done. Updates live as statuses change.

🗂

Filter by status

Filter the full task list to show only To do, In progress, Done, or All. Useful for focusing on what's next or reviewing what's complete.

💼

Relevant vault entries

Per-task tools
🔄

Status per task

Each task has a dropdown: To do, In progress, Done, N/A. Selecting Done strikes through the task. N/A is for tasks that don't apply to this estate.

📝

Notes per task

Attach a private note to any task — solicitor name, reference number, deadline, or anything relevant. Stored in the browser session.

Note: Executor task progress is stored in the browser session only — it resets when the tab is closed. This is by design (the portal contains sensitive vault data) but means executors should work through the checklist in one session or note progress externally. A future version will persist progress to the server.

Admin capabilities

Internal-only pages and controls for Digital Relative staff.

Death certificate review
📋

Manual review queue

Admin page at /admin-review showing all pending emergency access requests. Each row shows requester name, vault owner, submission date, and certificate type.

🪪

Certificate preview

Signed Supabase URL for each uploaded death certificate, viewable in a sandboxed iframe (PDF) or img tag (image). Access requires an admin token — not accessible to regular users.

✅

Manual approval

Admin can approve a pending request — setting manually_approved = true on the access request. The scheduler then processes it within the next cron run (subject to the 48-hour hold).

🔐

Token-gated access

The admin review page generates a time-limited, single-use review token rather than using a static admin token. The token is generated server-side and verified on every admin action. The review sandbox attribute prevents script execution in PDFs.

Security monitoring
🎭

Duress alerts

When a vault owner uses their duress PIN, an email is sent silently to both the owner's registered address and admin@security. The alert contains the owner's name, timestamp, and user ID.

📧

Monthly health emails

On the 1st of each month, the scheduler sends a vault health summary to each paid user: entry count, expired entries, unconfirmed beneficiaries, and a prompt to check in. Helps retain active users and reduce false-trigger risk.

Infrastructure
⚙️

Daily scheduler

The checkin-scheduler edge function runs daily via Supabase cron. Handles: check-in reminders (3/7/14 days overdue), SMS reminders, dead man's switch trigger, 48-hour hold processing, expiry reminders, and monthly health emails (1st of month only).

🚦

Rate limit monitoring

The rate_limits table logs all rate-limited actions (beneficiary access, emergency submit, MFA). A probabilistic TTL trigger cleans entries older than 1 hour to prevent unbounded growth.

🏗

Deployment

Git push to main triggers Vercel auto-deploy for the frontend. A GitHub Action deploys changed edge functions. Deploy script available at .\deploy.ps1 for local deployment.
