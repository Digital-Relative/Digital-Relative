1. Service summary

Digital Relative is a zero-knowledge digital legacy vault for UK families. Users store encrypted account credentials, secure notes, and personal documents for access by nominated beneficiaries in the event of death or incapacity. The service includes a configurable check-in system (dead man's switch), Onfido-verified emergency access, and a beneficiary portal with guided estate administration tools.

2. Architecture

2.1 Zero-knowledge encryption

All vault data is encrypted client-side before transmission. The server never holds encryption keys.

4.2 Multi-factor authentication

  • Email/password users: TOTP (authenticator app) or email OTP fallback
  • OAuth users (Google/Apple): bypass MFA — authentication delegated to provider
  • MFA unenrolment requires valid email OTP with 5-attempt lockout (service role, no AAL2 bypass)
  • Lost device: email OTP verification then service-role TOTP deletion

4.3 Emergency access

  • Mandatory 48-hour hold for all paths (death certificate, ID only, admin manual)
  • Owner notified with deny button during hold period
  • Onfido death certificate verification with magic-byte file validation
  • Per-IP rate limiting on submission (5 per hour)
  • Access requirement enforced per-beneficiary (death_certificate / id_only / trust_only)
  • grantVaultAccess filters by access_requirement — trust_only excluded

4.4 Additional security features

  • Duress PIN — secondary PIN shows decoy vault; silently alerts owner and admin
  • Trusted device — optional per-device PIN storage using device-derived encryption key
  • 8-hour absolute session — applies to all users including trusted devices
  • Device log — new device email alerts via Cloudflare geo headers
  • Shared link security — PBKDF2 PIN, optimistic lock on attempts, owner notification on access
  • Geographic access alerts — owner notified if vault accessed from outside UK
  • Vault entry versioning — 3 previous versions including title, category, and all encrypted fields
  • Recovery codes — 8 one-time PBKDF2-encrypted PIN recovery codes with fetch_count exfiltration detection
